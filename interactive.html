Understanding Path Planning Algorithms

This interactive demo allows you to compare different pathfinding algorithms. Each algorithm has unique characteristics in how it explores the search space and finds paths between start and goal positions.

A* uses a heuristic function to guide exploration toward the goal efficiently, balancing actual cost with estimated remaining distance. Dijkstra guarantees the shortest path by systematically exploring all directions without heuristic guidance. BFS explores level by level, finding the shortest path in unweighted graphs. DFS dives deep into one path before backtracking, which may not find the optimal path but uses less memory.

As algorithms run, light blue cells show explored nodes and dark blue shows the final path. Compare how many cells each algorithm explores to understand their efficiency differences.

Experiment Ideas

Straight Line

Place start and goal with no obstacles between them. Notice how few cells the algorithm explores when the path is direct.

Maze Navigation

Create complex mazes with narrow passages. Observe how A* finds the optimal route even through intricate obstacle patterns.

Multiple Paths

Create scenarios where several paths exist. A* will always find the shortest one, demonstrating its optimality guarantee.
